Esta es la pregunta que más nos hacen antes de iniciar un proyecto. Y la respuesta honesta es que depende. Pero hay criterios concretos para tomar esa decisión, y en este artículo te los vamos a dar.

El problema con decir simplemente "depende" es que no ayuda a nadie. Así que vamos a ser específicos: cuándo tiene sentido desarrollar algo propio, cuándo conviene usar lo que ya existe, y cómo evitar los errores más caros que vemos cometer a las empresas.

## ¿Qué es el software a medida?

El software a medida (o software personalizado) es una aplicación diseñada y desarrollada específicamente para las necesidades de tu empresa. No existe antes de que se construya. El código es tuyo, la lógica de negocio es tuya, y puedes modificarlo como quieras en el futuro.

Ejemplos: un sistema de gestión de proyectos hecho para el flujo específico de tu empresa de construcción, una plataforma de cotizaciones adaptada a tu modelo de precios, un e-commerce con lógica de descuentos compleja que ninguna plataforma estándar soporta.

## ¿Qué es el software genérico (o de plataforma)?

Son soluciones construidas para resolver problemas comunes de muchas empresas. Se compran como licencia o suscripción mensual. No tienes el código fuente; lo que tienes es acceso a la plataforma.

Ejemplos: Shopify para tiendas online, HubSpot para CRM y marketing, WordPress para sitios web, Notion para gestión de información, SAP Business One para ERP, QuickBooks para contabilidad.

## La comparación honesta

Antes de entrar en los criterios, veamos las diferencias clave entre ambos enfoques:

| Dimensión                    | Software a medida           | Plataforma / SaaS                            |
| ---------------------------- | --------------------------- | -------------------------------------------- |
| **Costo inicial**            | Alto ($5.000 - $100.000+)   | Bajo ($0 - $5.000)                           |
| **Costo mensual**            | Bajo (solo hosting)         | Medio-alto ($50 - $2.000/mes)                |
| **Tiempo de implementación** | 1-6 meses                   | Días o semanas                               |
| **Adaptación a tu proceso**  | Total                       | Parcial (debes adaptarte tú)                 |
| **Escalabilidad**            | Depende de la arquitectura  | Generalmente alta                            |
| **Riesgo de dependencia**    | Bajo (código es tuyo)       | Alto (si cierra la plataforma, pierdes todo) |
| **Mantenimiento**            | Tu responsabilidad          | Del proveedor                                |
| **Integraciones**            | Cualquier cosa técnicamente | Las que soporte la plataforma                |

## Cuándo el software a medida tiene sentido

### 1. Tu proceso es genuinamente único

Si el proceso que necesitas digitalizar tiene lógica que no existe en ninguna plataforma estándar, desarrollar tiene sentido. No porque sea "mejor" en abstracto, sino porque literalmente no hay otra opción.

Ejemplo real: una empresa de arriendo de maquinaria pesada con un modelo de precios que depende de factores climáticos, distancia, tipo de suelo y disponibilidad de operadores. Ningún software de arriendo estándar maneja esta lógica. La solución fue desarrollar un sistema de cotización a medida.

### 2. El software es tu producto o tu ventaja competitiva

Si estás construyendo una startup tecnológica, una plataforma de servicios o cualquier negocio donde el software ES el producto, no puedes construirlo sobre la plataforma de otra empresa.

### 3. El volumen hace que las licencias sean caras

Las plataformas SaaS escalan sus precios con el uso. Cuando llegas a cierto volumen, desarrollar propio puede ser más barato que seguir pagando licencias.

Ejemplo típico: una empresa con 5.000 transacciones mensuales en una plataforma SaaS avanzada puede estar pagando entre $400-$1.500 USD/mes sumando suscripción, apps y comisiones. Un e-commerce propio puede operar con $80-$300 USD/mes de hosting e infraestructura. El break-even se alcanza en un rango de 1,5 a 4 años según el proveedor y volumen.

### 4. Necesitas integraciones profundas con sistemas existentes

Si necesitas conectar con un sistema antiguo (legacy), con sistemas propietarios de tu industria o con SAP/Oracle de una forma no estándar, el desarrollo a medida te da el control que las plataformas no permiten.

### 5. Tienes requisitos de seguridad o cumplimiento normativo especiales

Datos médicos (MINSAL), datos financieros (CMF), o requisitos de seguridad del Estado chileno pueden obligarte a tener control total sobre dónde y cómo se almacenan los datos. Muchas plataformas SaaS almacenan datos en servidores fuera de Chile, lo que puede ser un problema.

## Cuándo conviene usar una plataforma

### 1. El problema es estándar y la plataforma lo resuelve bien

Si necesitas un sitio web corporativo y WordPress/Webflow lo hace perfectamente, ¿por qué desarrollar desde cero? Usar lo que ya existe no es una mala decisión; es una decisión inteligente.

### 2. Necesitas resultados en semanas, no en meses

Las plataformas permiten lanzar en días. Si el tiempo de mercado es crítico (campaña de temporada, lanzamiento de producto), usar una plataforma existente tiene sentido aunque a largo plazo no sea la solución ideal.

### 3. No tienes equipo técnico para mantener código propio

El software a medida requiere mantenimiento constante: actualizaciones de seguridad, corrección de bugs, nuevas funcionalidades. Si no tienes un equipo técnico o presupuesto para contratar uno, el riesgo de quedarse con software sin soporte es alto.

### 4. Estás validando una idea

Si estás en etapa de validar si hay demanda para tu producto, gastar $50.000 USD en desarrollo es un error. Primero valida con herramientas existentes; cuando tengas tracción real, invierte en desarrollo propio.

### 5. Presupuesto limitado y funcionalidades estándar

Una PYME que necesita gestionar sus clientes con un CRM básico puede usar HubSpot Free, Zoho o Pipedrive por $0-$50/mes. Desarrollar un CRM propio para el mismo caso no tiene sentido económico.

## El error más caro que vemos: el falso ahorro

El error más común es creer que "desarrollar propio es más barato porque no hay mensualidad". Esto ignora:

- **Costo de desarrollo inicial**: puede ser 10-100 veces el costo de la primera mensualidad de una plataforma.
- **Tiempo hasta tener algo funcionando**: meses vs. días.
- **Mantenimiento**: actualizaciones, seguridad, bugs. Esto no desaparece.
- **Costo de oportunidad**: mientras se desarrolla, tu equipo no puede trabajar en el negocio.

La comparación correcta es el **costo total de propiedad en 3-5 años**, no el costo del mes 1.

## El otro error: quedarse atrapado en una plataforma

El error opuesto es elegir una plataforma sin considerar los riesgos:

- **Vendor lock-in**: migrar desde Shopify cuando tienes 10.000 productos y años de datos es complejo y caro.
- **Aumento de precios**: Shopify aumentó sus precios un 33% en 2023. Si dependes completamente de ellos, no tienes opción.
- **Limitaciones que bloquean el crecimiento**: muchas empresas llegan a un punto donde la plataforma ya no puede hacer lo que necesitan, y migrar en ese punto es mucho más caro que haber desarrollado propio desde el inicio.

## Costo total de propiedad: el cálculo que pocos hacen

La comparación más honesta no es precio inicial vs. mensualidad. Es el **costo total en 3 años**, incluyendo todos los factores.

Tomemos un ejemplo concreto: una tienda online mediana con 500 productos y ~200 pedidos mensuales.

> **Nota**: los rangos de abajo reflejan el mercado real, no un solo proveedor. Según la plataforma o agencia elegida, los valores pueden estar en el extremo inferior o superior del rango indicado.

### Opción A: Plataforma SaaS e-commerce nivel avanzado (ej: Shopify, WooCommerce cloud, VTEX Go, Jumpseller)

| Concepto | Año 1 | Año 2 | Año 3 |
| --- | --- | --- | --- |
| Suscripción de la plataforma | $1.800 - $4.800 USD | $1.800 - $4.800 USD | $1.800 - $4.800 USD |
| Apps/plugins adicionales | $600 - $2.400 USD | $600 - $2.400 USD | $600 - $2.400 USD |
| Customización inicial | $1.000 - $5.000 USD | - | - |
| Comisiones por transacción (0,3%-2% según plan) | $720 - $4.800 USD | $720 - $4.800 USD | $720 - $4.800 USD |
| **Total acumulado (rango)** | **$4.120 - $17.000 USD** | **$7.240 - $29.000 USD** | **$10.360 - $41.000 USD** |

### Opción B: E-commerce a medida

| Concepto | Año 1 | Año 2 | Año 3 |
| --- | --- | --- | --- |
| Desarrollo inicial | $10.000 - $30.000 USD | - | - |
| Hosting + infraestructura | $600 - $2.400 USD | $600 - $2.400 USD | $600 - $2.400 USD |
| Mantenimiento (15-20% del dev/año) | $1.500 - $6.000 USD | $1.500 - $6.000 USD | $1.500 - $6.000 USD |
| Nuevas funcionalidades | $2.000 - $6.000 USD | $1.500 - $4.000 USD | $1.000 - $3.000 USD |
| **Total acumulado (rango)** | **$14.100 - $44.400 USD** | **$18.700 - $56.800 USD** | **$22.800 - $68.200 USD** |

> **Qué dice la comparación**: en el extremo inferior del rango (empresa pequeña, bajo volumen de transacciones), la plataforma SaaS casi siempre sale más barata en los primeros 3 años. En el extremo superior (alto volumen, muchas comisiones, muchos plugins), el software a medida puede empezar a ser competitivo desde el año 2.

**¿Cuándo cambia este cálculo a favor del desarrollo propio?**
- Cuando el volumen de pedidos supera los 1.500-2.000/mes y las comisiones por transacción se vuelven significativas.
- Cuando el costo mensual de apps/plugins supera el 30% del costo de mantenimiento de un sistema propio.
- Cuando la plataforma no puede implementar una funcionalidad crítica para el negocio sin un desarrollo tan costoso como hacerlo desde cero.
- Cuando la plataforma aumenta sus precios o cambia sus condiciones (algo que ocurre con frecuencia en plataformas SaaS maduras).



Usa estas preguntas para tomar la decisión:

**1. ¿Existe una plataforma que resuelva el 80% o más de lo que necesito?**
→ Sí: evalúa la plataforma primero.
→ No: considera desarrollo a medida.

**2. ¿Cuánto volumen tendrás en 3 años?**
→ Calcula el costo de la plataforma a ese volumen. Si es mayor que el desarrollo + mantenimiento, el desarrollo propio puede ser mejor.

**3. ¿Tienes equipo para mantener software propio?**
→ No: asegúrate de tener un proveedor de soporte antes de comprometerte con desarrollo.

**4. ¿El proceso es estándar o tiene lógica única?**
→ Estándar: plataforma.
→ Único: desarrollo a medida.

**5. ¿Cuánto tiempo tienes?**
→ Urgente (menos de 1 mes): plataforma.
→ Mediano plazo (3-6 meses): desarrollo puede ser viable.

## ¿Y la opción híbrida?

Muchas veces la mejor respuesta es una combinación: usar una plataforma para lo estándar y desarrollar solo lo que realmente es único.

Ejemplo: una empresa de servicios que usa HubSpot para CRM (plataforma) y desarrolla a medida solo el módulo de cotización y seguimiento de proyectos (que tiene lógica única). Esto optimiza costos sin comprometer las funcionalidades diferenciadas.
